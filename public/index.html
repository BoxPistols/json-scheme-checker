認証情報の保存について

Basic認証のユーザー名とパスワードをブラウザに保存する方法を選択できます。各オプションの特徴を理解して、ご自身の環境に合った方法を選択してください。

保存方法 セキュリティ 利便性 保存期間
保存しない 高 低 なし
タブを閉じるまで 中 中 タブを閉じるまで
24時間保存 低 高 24時間
永続保存 極低 最高 期限なし

各オプションの詳細

保存しない（最もセキュア）

ページをリロードするたびに再入力が必要です。セキュリティを最優先する場合に推奨します。

タブを閉じるまで保存（推奨）

ブラウザのタブを閉じると自動的に削除されます。セキュリティと利便性のバランスが取れた推奨オプションです。認証情報は暗号化されて保存されます。

24時間保存（利便性重視）

24時間後に自動的に削除されます。頻繁にアクセスする場合に便利ですが、信頼できるデバイスでのみ使用してください。認証情報は暗号化されて保存されます。

永続保存（期限なし、最もリスク高）

期限なしで永続的に保存されます。手動でクリアするまで認証情報が残り続けます。最も利便性が高い一方、セキュリティリスクも最大です。個人デバイスで、かつ重要度の低い認証情報の場合のみ使用を検討してください。認証情報は暗号化されて保存されます。

注意: 暗号化はMyAPIツールで見られないようにするための簡易的なものです。完全なセキュリティを保証するものではありません。重要な認証情報は「保存しない」を選択することを強く推奨します。

Schema Viewer & AI Advisor - 使い方ガイド

このアプリでできること

Web構造化データの可視化から、AIによるSEO最適化支援まで、Webサイトの品質向上を総合的にサポートします。

Schema Viewer

JSON-LD構造化データの抽出・可視化。テーブル形式とJSON形式で見やすく表示。

AI分析

求人票の最適化、ブログ記事のレビュー、WebページのSEO/EEAT分析をAIが支援。

SEO最適化

HTML構造、メタタグ、SNS対応を自動分析。改善ポイントを具体的に提示。

Schema Viewer機能

JSON-LD構造化データの抽出と可視化

WebページからJSON-LD形式の構造化データを自動抽出し、分かりやすく表示します。

テーブル表示

プロパティ名、値、型を階層構造で見やすく表示。ネストされたオブジェクトも展開・折りたたみ可能。

JSON表示

生のJSON形式でシンタックスハイライト付き表示。開発者向けの詳細確認に最適。

ワンクリックコピー

各スキーマをクリップボードに即座にコピー。他のツールとの連携もスムーズ。

AI Advisor機能

AIがページ内容を分析し、具体的な改善提案を提供します。各Advisorは特定の目的に最適化されています。

JobPosting Advisor

対象：求人票（JobPostingスキーマ検出時）

分析内容：

  • 採用側視点：応募を増やすための改善提案
  • 応募者視点：求人情報の明確性と魅力度分析
  • エージェント視点：開発現場と技術要件を可視化、戦略的マッチングを支援
  • 構造化データの最適化アドバイス

Blog Reviewer

対象：ブログ記事（Articleスキーマ検出時）

分析内容：

  • 記事の構成と読みやすさの評価
  • SEO観点からの改善提案
  • EEAT（専門性・権威性・信頼性）の評価
  • 構造化データの最適化

Web Advisor

対象：汎用Webページ（スキーマ無し、またはWebPageのみ）

分析内容：

  • SEO最適化提案（タイトル、メタタグ、見出し構造）
  • EEAT評価と改善アドバイス
  • アクセシビリティ評価
  • 優先対応事項の提示

SEO分析機能

6つのタブでWebページを多角的に分析し、改善ポイントを明確化します。

タブ 分析内容
Schema JSON-LD構造化データの詳細表示。テーブル/JSON形式で切り替え可能。
HTML構造 HTML要素数、見出しタグ（h1〜h6）の使用状況、ページ構造の分析。
概要・メタタグ SEOスコア、タイトルタグ、メタディスクリプション、Robotsタグの検証。
SNS Open Graph（Facebook/LinkedIn）、Twitter Cardsの設定確認。
改善ガイダンス 検出された問題点と具体的な改善方法を優先度順に表示。

基本操作

1. URLを入力して分析開始

  1. 画面上部のURL入力欄に、分析したいWebページのURLを貼り付けます
  2. 「取得」ボタンをクリック、またはEnterキーを押します
  3. 数秒後、分析結果が表示されます

キーボードショートカット： Cmd+K（Mac）またはCtrl+K（Windows）でURL入力欄に素早くフォーカスできます。

2. Basic認証が必要なサイトの分析

ステージング環境などパスワード保護されたサイトも分析できます。

  1. 「Basic認証が必要な場合」セクションを展開
  2. ユーザー名とパスワードを入力
  3. 認証情報の保存方法を選択（推奨：タブを閉じるまで保存）
  4. URLを入力して分析を実行

セキュリティ： 認証情報はブラウザのみに保存され、サーバーには送信されません。重要な認証情報は「保存しない」を選択してください。

3. サンプルリンクの管理

よく使うURLを登録しておけば、ワンクリックで分析できます。

追加：「+ 追加」ボタンからURL名とアドレスを登録
編集：各URLの「編集」ボタンから情報を変更
削除：「削除」ボタンで不要なURLを削除
並び替え：URLタグをドラッグ＆ドロップで順序変更

Robots メタタグの設定方法

何をすべきか？

HTMLの <head> タグ内に以下のコードを追加してください。この対応でGoogleが正しくページを認識し、検索結果に表示できるようになります。

コピー可能な実装例

該当するセクションをコピーして使用してください

1. Static HTML（HTMLファイル直接編集）

<head> <meta name="robots" content="index,follow"> </head>

2. Vue / Nuxt.js

export default { head() { return { meta: [ { name: 'robots', content: 'index,follow' } ] } } }

3. React / Next.js

import Head from 'next/head'; export default function Home() { return ( <Head> <meta name="robots" content="index,follow" /> </Head> ) }

各値の意味

index Google が このページを検索対象に含めてもいい
follow ページ内のリンクをたどってもいい
noindex このページを検索結果から除外する
nofollow リンクをたどらない（ページは表示されるが）

推奨設定

ほとんどのWebサイトは index,follow が推奨です。この設定により、Googleはページを検索対象にでき、ページ内のリンクをたどってサイト全体をクロールできます。

確認方法

設定後、このツールで再度URLを入力すると、Robots メタタグが正しく設定されたか確認できます。

Open Graph タグの設定方法

何をすべきか？

HTMLの <head> タグ内に Open Graph タグを追加してください。この対応でページが Facebook、LinkedIn、Slack などで共有されたときに、タイトル、説明、画像が正しく表示されます。

必須プロパティ

og:title ページのタイトル（60文字程度）
og:description ページの説明（160文字程度）
og:image 共有時に表示される画像（推奨: 1200x630px）
og:url ページの正規URL
og:type ページタイプ（例: website, article など）

コピー可能な実装例

該当するセクションをコピーして使用してください

1. Static HTML（HTMLファイル直接編集）

<head> <meta property="og:title" content="ページタイトル"> <meta property="og:description" content="ページの説明文"> <meta property="og:image" content="https://example.com/image.jpg"> <meta property="og:url" content="https://example.com/page"> <meta property="og:type" content="website"> </head>

2. Vue / Nuxt.js

export default { head() { return { meta: [ { property: 'og:title', content: 'ページタイトル' }, { property: 'og:description', content: 'ページの説明文' }, { property: 'og:image', content: 'https://example.com/image.jpg' }, { property: 'og:url', content: 'https://example.com/page' }, { property: 'og:type', content: 'website' } ] } } }

3. React / Next.js

import Head from 'next/head'; export default function Page() { return ( <Head> <meta property="og:title" content="ページタイトル" /> <meta property="og:description" content="ページの説明文" /> <meta property="og:image" content="https://example.com/image.jpg" /> <meta property="og:url" content="https://example.com/page" /> <meta property="og:type" content="website" /> </Head> ) }

og:type の値

website 一般的なウェブサイト（デフォルト）
article ブログ記事やニュース
product 商品ページ
video.movie 動画コンテンツ

ヒント

og:image は最も重要です。高品質で視覚的に目立つ画像を選ぶと、シェア時のクリック率が大幅に向上します。推奨サイズは 1200x630px です。

確認方法

設定後、このツールで再度URLを入力すると、Open Graph タグが正しく設定されたか確認できます。また、OGP確認ツールなどでも確認できます。

Twitter Card の設定方法

何をすべきか？

HTMLの <head> タグ内に Twitter Card のメタタグを追加してください。この対応でTwitterでページがシェアされたときに、美しいカード形式で表示されます。

Twitter Card の種類

summary シンプルな形式（タイトル + 説明 + アイコン）
summary_large_image 大きな画像付き形式（推奨）
app モバイルアプリのプロモーション
player 動画やオーディオ再生用

コピー可能な実装例

該当するセクションをコピーして使用してください

基本的な設定（全ページ共通）

<!-- Twitter Card 基本設定 --> <meta name="twitter:card" content="summary_large_image"> <meta name="twitter:site" content="@あなたのTwitterID"> <meta name="twitter:creator" content="@あなたのTwitterID">

1. Static HTML（HTMLファイル直接編集）

<head> <!-- Twitter Card --> <meta name="twitter:card" content="summary_large_image"> <meta name="twitter:title" content="ページのタイトル"> <meta name="twitter:description" content="ページの説明（140字以内推奨）"> <meta name="twitter:image" content="https://example.com/image.jpg"> <meta name="twitter:site" content="@YourTwitterID"> <meta name="twitter:creator" content="@YourTwitterID"> </head>

2. Vue / Nuxt.js

export default { head() { return { meta: [ { name: 'twitter:card', content: 'summary_large_image' }, { name: 'twitter:title', content: 'ページのタイトル' }, { name: 'twitter:description', content: 'ページの説明' }, { name: 'twitter:image', content: 'https://example.com/image.jpg' }, { name: 'twitter:site', content: '@YourTwitterID' }, { name: 'twitter:creator', content: '@YourTwitterID' } ] } } }

3. React / Next.js

import Head from 'next/head'; export default function Home() { return ( <Head> <meta name="twitter:card" content="summary_large_image" /> <meta name="twitter:title" content="ページのタイトル" /> <meta name="twitter:description" content="ページの説明" /> <meta name="twitter:image" content="https://example.com/image.jpg" /> <meta name="twitter:site" content="@YourTwitterID" /> <meta name="twitter:creator" content="@YourTwitterID" /> </Head> ) }

各フィールドの説明

twitter:card カード形式（summary_large_image 推奨）
twitter:title シェアされた時に表示されるタイトル（70字以内）
twitter:description ページの説明（140字以内推奨）
twitter:image シェア時に表示される画像URL（1200x630px推奨）
twitter:site サイトの Twitter アカウント（オプション）
twitter:creator 記事著者の Twitter アカウント（オプション）

実装のコツ

  • twitter:image は必ず HTTPS で指定
  • 画像サイズは 1200x630px（または 1:1、2:1 のアスペクト比）
  • summary_large_image が最も効果的
  • 動的に内容が変わるページの場合、各ページで異なる値を設定

テスト方法

Twitter Card Validator にURLを入力して、プレビューを確認できます。

Open Graph との関連性

Open Graph（og:title, og:description, og:image）も同時に設定することで、Facebook や LinkedIn でも美しく表示されます。

スキーマ構造化データ学習ガイド

JSON-LD スキーマとは

JSON-LD（JSON for Linking Data）は、Webページの内容をGoogleなどの検索エンジンに正しく理解させるためのコード形式です。

例えば、JobPosting スキーマを使うと、採用情報が「求人検索」の結果に表示される可能性があります。

JobPosting スキーマ（採用情報）

用途: 求人情報をGoogleに正しく伝える

メリット: Google 求人検索に表示される可能性がある

最小限の実装例

<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "JobPosting", "title": "エンジニア募集", "description": "Javaエンジニアを募集しています", "datePosted": "2025-01-22", "validThrough": "2025-03-22", "hiringOrganization": { "@type": "Organization", "name": "会社名" }, "jobLocation": { "@type": "Place", "address": { "@type": "PostalAddress", "addressCountry": "JP" } }, "employmentType": "FULL_TIME", "baseSalary": { "@type": "MonetaryAmount", "currency": "JPY", "value": { "@type": "QuantitativeValue", "minValue": 600000, "maxValue": 800000, "unitText": "MONTH" } } } </script>

Article スキーマ（ブログ記事）

用途: ブログ記事情報をGoogleに伝える

メリット: リッチスニペット表示、Google 検索に優先的に表示される可能性

実装例

<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Article", "headline": "記事のタイトル", "description": "記事の説明", "image": "https://example.com/image.jpg", "datePublished": "2025-01-22", "dateModified": "2025-01-23", "author": { "@type": "Person", "name": "著者名" }, "publisher": { "@type": "Organization", "name": "メディア名", "logo": { "@type": "ImageObject", "url": "https://example.com/logo.jpg" } } } </script>

Organization スキーマ（組織情報）

用途: 企業・組織情報をGoogleに伝える

メリット: ナレッジパネルに企業情報が表示される可能性

実装例

<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Organization", "name": "会社名", "url": "https://example.com", "logo": "https://example.com/logo.png", "description": "会社の説明", "sameAs": [ "https://twitter.com/company", "https://www.facebook.com/company" ], "contactPoint": { "@type": "ContactPoint", "contactType": "Customer Support", "email": "support@example.com" } } </script>

実装のコツ

  • JSONは正確に記述する（カンマやコロンの誤字に注意）
  • Google 構造化データテストツールで検証する
  • このツールでもスキーマの正確性をチェックできます
