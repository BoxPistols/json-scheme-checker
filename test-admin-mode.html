隠しモード機能 - 動作確認テスト（user パラメータ版）

テスト1: 通常アクセス（隠しモードOFF）

通常のURLでアクセスすると、ベータ機能ボタンは表示されません。

通常アクセスでテスト

期待される動作:

  • 「コンテンツをアップロード」ボタンが非表示
  • 「My Skill Sheet」ボタンが非表示
  • 「レジュメビルダー」ボタンが非表示
  • URL入力欄の下に何も表示されない

テスト2: 隠しモード（user=file）

URLに ?user=file を追加すると、コンテンツアップロード機能が表示されます。

user=file でテスト

期待される動作:

  • 「コンテンツをアップロード BETA」ボタンのみ表示される
  • 他のベータ機能ボタンは非表示
  • ボタンをクリックすると、アップロードモーダルが開く

テスト3: 隠しモード（user=skill）

URLに ?user=skill を追加すると、My Skill Sheet機能が表示されます。

user=skill でテスト

期待される動作:

  • 「My Skill Sheet BETA」ボタンのみ表示される
  • 他のベータ機能ボタンは非表示
  • ボタンをクリックすると、/skill?user=skill に遷移する

テスト4: 隠しモード（user=resume）

URLに ?user=resume を追加すると、レジュメビルダー機能が表示されます。

user=resume でテスト

期待される動作:

  • 「レジュメビルダー BETA」ボタンのみ表示される
  • 他のベータ機能ボタンは非表示
  • ボタンをクリックすると、レジュメビルダーモーダルが開く

テスト5: 無効なパラメータ

user=admin や user=test など、定義されていない値では機能が有効にならないことを確認します。

user=admin でテスト user=test でテスト

期待される動作:

  • どちらの場合も、すべてのベータ機能ボタンは非表示
  • user=file、user=skill、user=resume のみが有効

実装詳細

隠しモード機能は以下のように実装されています:

  • トリガー: URLクエリパラメータ ?user=xxx
  • 対象機能:
    • ?user=file: コンテンツアップロード
    • ?user=skill: My Skill Sheet
    • ?user=resume: レジュメビルダー
  • デフォルト動作: ボタンは display: none で完全非表示
  • セキュリティ: 各機能は独立した鍵で管理され、推測されにくい

使用方法

本番環境（Vercel）でも同様に動作します:

https://json-ld-view.vercel.app/?user=file https://json-ld-view.vercel.app/?user=skill https://json-ld-view.vercel.app/?user=resume

これらのURLをブックマークしておくと、常に隠しモードでアクセスできます。
