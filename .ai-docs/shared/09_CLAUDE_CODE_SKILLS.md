# Claude Code Skills ガイド

## 概要

Claude Code Skillsは、プロジェクト特有のタスクを効率化するためのカスタム機能です。`.claude/skills/`ディレクトリに配置されたMarkdownファイルがスキルとして認識されます。

## 利用可能なスキル

### 1. api-check
**目的**: API仕様の一貫性と品質をチェック

**使用タイミング**:
- 新しいAPIエンドポイントを追加した後
- 既存APIの大規模な変更を行った後
- 定期的な品質チェック

**実行方法**:
```
api-check
```

**チェック項目**:
- エンドポイント構造の確認
- CORS設定の確認
- エラーハンドリングの確認
- 入力検証の確認
- レート制限の確認
- レスポンス形式の確認

### 2. deploy-check
**目的**: Vercelへのデプロイ前に必要な全ての確認を実行

**使用タイミング**:
- 本番環境へのデプロイ前（必須）
- プルリクエストのマージ前
- 重要な機能追加後

**実行方法**:
```
deploy-check
```

**チェック項目**:
- 環境設定の確認
- 依存関係の確認
- ビルドとテストの確認
- Vercel設定の確認
- セキュリティチェック
- ドキュメントの確認
- パフォーマンスチェック

### 3. code-review
**目的**: コード変更に対する包括的なレビューを実行

**使用タイミング**:
- プルリクエストを作成した後
- 大きな機能を実装した後
- コードレビューの準備として

**実行方法**:
```
code-review
```

**レビュー観点**:
- コード品質
- エラーハンドリング
- セキュリティ
- パフォーマンス
- テスタビリティ
- ベストプラクティス
- ドキュメント

## スキルの作成方法

### 基本構造

``markdown
# スキル名

スキルの説明

## 実行内容
[詳細な指示]

## 実行手順
[ステップバイステップの手順]

## 出力形式
[期待される出力の形式]

## 注意事項
[特記事項]
```

### 命名規則
- ファイル名: `kebab-case.md`
- スキル名（呼び出し時）: ファイル名から`.md`を除いたもの

### ベストプラクティス
1. **具体的な指示**: 曖昧な表現を避け、具体的なアクションを記述
2. **構造化された出力**: 出力形式を明確に定義
3. **プロジェクト特有のルール**: プロジェクトのコーディング規約を反映
4. **実行可能な提案**: 実装可能な具体的な改善提案を促す

## トラブルシューティング

### スキルが認識されない
- `.claude/skills/`ディレクトリが存在することを確認
- ファイルの拡張子が`.md`であることを確認
- ファイル名にスペースや特殊文字が含まれていないことを確認

### スキルの実行が失敗する
- プロンプトの指示が明確か確認
- 必要なファイルが存在するか確認
- 権限の問題がないか確認
