## 1. ペルソナ定義

> **重要**: 人材紹介エージェント（ペルソナ1-4）は本ツールの最重要ターゲットユーザーです。
> JobPosting Advisor、スキルシートレビュー、マッチング機能は彼らの業務効率化に直結します。

### ペルソナ1: キャリアアドバイザー（CA） - 高橋 理恵（28歳）【最重要】

**基本情報**

| 項目             | 内容                                               |
| ---------------- | -------------------------------------------------- |
| 職種             | ITエンジニア専門人材紹介会社のキャリアアドバイザー |
| 経験年数         | CA歴3年（元SE）                                    |
| 担当領域         | 求職者側（候補者のキャリア支援）                   |
| 所在地           | 東京都港区                                         |
| 使用デバイス     | Windows PC、iPhone                                 |
| 月間担当候補者数 | 40-50名                                            |
| 月間成約目標     | 5-8名                                              |

**目標と動機**

- 候補者のスキルシートを改善して書類通過率を上げたい
- 候補者と求人のマッチング精度を向上させたい
- 面談効率を上げて、より多くの候補者を支援したい
- 候補者への具体的なアドバイスの質を高めたい

**課題と不満**

- スキルシートの添削に時間がかかりすぎる（1件30分以上）
- 候補者の経歴をどうアピールすべきか、毎回悩む
- 求人票を見ても、本当にマッチする候補者か判断が難しい
- 新人CAへの教育が属人的になっている

**行動パターン**

- 毎朝、担当候補者の進捗確認
- 1日3-5件の候補者面談
- 面談後にスキルシート添削・作成
- 夕方にRAと候補者の推薦打ち合わせ
- 週1で求人情報の棚卸し

**技術環境**

- 社内ATS（候補者管理システム）
- Excel/Google Sheets（スキルシート作成）
- Salesforce（CRM）
- Zoom/Google Meet（オンライン面談）
- LINE/メール（候補者連絡）

**このツールの活用シーン**

1. **スキルシートレビュー**: 候補者から受け取ったスキルシートをアップロードし、改善点を即座に把握
2. **マッチング分析**: 候補者のスキルシートと求人票をマッチングし、推薦可否を客観的に判断
3. **フィードバック作成**: AIの分析結果を候補者へのアドバイスに活用

**KPI への貢献**

| KPI                  | 現状    | ツール活用後の期待 |
| -------------------- | ------- | ------------------ |
| スキルシート添削時間 | 30分/件 | 10分/件            |
| 書類通過率           | 25%     | 35%                |
| 候補者満足度         | 3.5/5   | 4.2/5              |

**引用**

> 「スキルシートの添削が一番時間かかるんです。AIが改善点を出してくれるなら、面談にもっと時間を使えます」

### ペルソナ2: リクルーティングアドバイザー（RA） - 中村 大輔（34歳）【最重要】

**基本情報**

| 項目         | 内容                                                       |
| ------------ | ---------------------------------------------------------- |
| 職種         | ITエンジニア専門人材紹介会社のリクルーティングアドバイザー |
| 経験年数     | RA歴5年（人材業界一筋）                                    |
| 担当領域     | 企業側（求人開拓・採用支援）                               |
| 所在地       | 東京都渋谷区                                               |
| 使用デバイス | MacBook Pro、iPhone                                        |
| 担当企業数   | 30-40社                                                    |
| 月間求人数   | 80-100件                                                   |

**目標と動機**

- クライアント企業の求人票を改善して応募率を上げたい
- 競合他社の求人と差別化するポイントを見つけたい
- 企業への提案資料を効率的に作成したい
- 求人票のGoogle for Jobs表示を最適化したい

**課題と不満**

- 求人票の改善提案に説得力がない（感覚的になりがち）
- 競合他社の求人分析に時間がかかる
- クライアントに技術的なことを説明するのが難しい
- 求人票の品質にばらつきがある

- 毎朝、担当企業の採用状況確認
- 1日2-3件の企業訪問/オンライン商談

- 週次で企業へのレポート作成

**技術環境**

- 社内ATS（求人管理システム）
- Salesforce（CRM）
- PowerPoint/Google Slides（提案資料）
- 各求人媒体管理画面
- ChatGPT（文章作成補助）

**このツールの活用シーン**

1. **JobPosting Advisor**: 求人ページの構造化データを分析し、改善提案を企業に報告
2. **競合分析**: 競合他社の求人ページを分析し、差別化ポイントを抽出
3. **SEO改善提案**: 企業の採用サイト全体のSEO改善を提案

**KPI への貢献**

| KPI              | 現状       | ツール活用後の期待 |
| ---------------- | ---------- | ------------------ |
| 求人あたり応募数 | 8件/月     | 12件/月            |
| 企業提案採用率   | 40%        | 55%                |
| 競合分析時間     | 2時間/企業 | 30分/企業          |

**引用**

> 「求人票の改善って、『もっと魅力的に』としか言えなかった。AIが具体的に指摘してくれるなら、企業への説得力が全然違う」

### ペルソナ3: 両面型エージェント - 渡辺 健太（31歳）【最重要】

**基本情報**

| 項目             | 内容                                           |
| ---------------- | ---------------------------------------------- |
| 職種             | ブティック型人材紹介会社の両面型コンサルタント |
| 経験年数         | 人材紹介6年（両面型3年）                       |
| 担当領域         | 企業側・求職者側の両方                         |
| 所在地           | 大阪府大阪市                                   |
| 使用デバイス     | MacBook Air、iPhone                            |
| 担当企業数       | 15-20社                                        |
| 月間担当候補者数 | 20-30名                                        |

**目標と動機**

- 企業と候補者の両方を深く理解してマッチング精度を最大化したい
- 1人で完結する効率的なワークフローを構築したい
- 成約までのリードタイムを短縮したい
- 高単価案件（年収800万以上）の成約を増やしたい

**課題と不満**

- 両面を担当するため、とにかく時間が足りない
- 候補者スキルと求人要件のマッチング判断が感覚的
- 求人票の品質管理と候補者対応を両立するのが大変
- 属人的な判断に頼りすぎている

**行動パターン**

- 午前: 企業対応（求人確認、推薦候補者の報告）
- 午後: 候補者対応（面談、スキルシート作成）
- 夕方: マッチング検討、推薦準備
- 夜: 候補者からの問い合わせ対応
- 週末: 翌週の商談準備

**技術環境**

- Notion（個人タスク管理）
- HubSpot（CRM）
- Excel/Google Sheets
- Calendly（日程調整）
- Slack（社内連絡）

**このツールの活用シーン**

1. **統合活用**: 求人分析→候補者スキルシート改善→マッチング分析を一気通貫で実施
2. **時短**: AI分析結果をそのまま企業・候補者への連絡に活用
3. **品質標準化**: 属人的な判断をAI分析で補完

**KPI への貢献**

| KPI           | 現状   | ツール活用後の期待 |
| ------------- | ------ | ------------------ |
| 月間成約数    | 3件    | 5件                |
| 推薦→内定率   | 20%    | 30%                |
| 1件あたり工数 | 15時間 | 10時間             |

**引用**

> 「両面やっていると、本当に時間がない。求人分析からマッチングまで、1つのツールで完結できるなら最高です」
