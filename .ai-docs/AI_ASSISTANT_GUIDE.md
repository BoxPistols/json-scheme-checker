# AIアシスタント活用ガイド

このプロジェクトでは、開発効率を最大化するために、AIアシスタント（Claude Code / Gemini）を積極的に活用しています。
AIは、単純なコード生成だけでなく、プロジェクトの文脈を理解し、レビューやドキュメント検索といった高度なタスクを自動化します。

このガイドは、プロジェクトで利用可能なAI関連機能の全体像を理解するための入り口です。

## 1. 全体像：MCP, Skills, SubAgent

AIアシスタントの能力は、主に3つの要素で構成されています。

- **MCP (Model Context Protocol):**
  - AIが外部ツール（ドキュメント、API、開発ツール）と対話するための「言語」や「API」のようなものです。
- **Skills:**
  - 特定のタスク（例: コードレビュー）をAIに実行させるための、再利用可能な「指示書（プロンプト）」です。
- **SubAgents:**
  - スキルを実行する際に、AIに特定の専門家（例: コードレビュアー）として振る舞わせるための「役割設定」です。

これらの関係性は以下のようになります。
`SubAgent（役割）` が `Skill（指示書）` に基づいてタスクを実行し、必要に応じて `MCP（外部ツール）` を利用して情報を収集・操作します。

---

## 2. MCP (Model Context Protocol)

### 概要

MCPは、AIモデルが外部のツールやサービスと対話するための標準プロトコルです。これにより、AIは単独では不可能な、より実践的なタスクを実行できます。

- **例:**
  - Chrome DevToolsと連携してWebページのデバッグを行う。
  - データベースに接続してクエリを実行する。
  - プロジェクト専用のドキュメントサーバーから最新情報を取得する。

### このプロジェクトで利用可能なMCPサーバー

#### `context7`

- **目的:** npmパッケージのライブラリドキュメントやコード例をリアルタイムで取得します。
- **設定:** `.mcp.json`
- **使い方:** `「Context7でExpressのストリーミング方法を教えて」` のように、自然言語で質問します。
- **詳細:** [10_MCP_USAGE_GUIDE.md](./shared/10_MCP_USAGE_GUIDE.md)

## 3. Skills

### 概要

Skillsは、特定の定型タスクをAIに一貫した品質で実行させるための、再利用可能なプロンプト（指示書）です。

- **目的:** 「コードレビュー」「デプロイ前チェック」のような複雑なタスクを、誰が実行しても同じ基準で行えるように標準化します。
- **格納場所:** `/.claude/skills/` ディレクトリ

### 主な利用可能スキル

- `code-review`: コードの品質、バグ、セキュリティなどを包括的にレビューします。
- `api-check`: APIエンドポイントの設計や実装に一貫性があるかチェックします。
- `deploy-check`: Vercelへのデプロイ前に設定やビルドに問題がないか確認します。

### 使い方

AIアシスタントのチャットで、単にスキル名を入力して実行します。

```
code-review
```

- **詳細:** [09_CLAUDE_CODE_SKILLS.md](./shared/09_CLAUDE_CODE_SKILLS.md)

## 4. SubAgents

### 概要

SubAgentは、AIに特定のタスクを実行させる際の「役割（ペルソナ）」を定義する、より内部的な概念です。

- **例:** `code-review` スキルを実行する際、AIは「経験豊富なシニアソフトウェアエンジニア」というSubAgentとして振る舞うよう指示されています。
- **目的:** タスクの目的に応じて、AIの応答のトーン、視点、専門性を最適化します。

通常、開発者がSubAgentを直接意識する必要はありません。これは主にSkillsの内部で活用される仕組みです。
